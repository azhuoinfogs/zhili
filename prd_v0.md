1. 产品概述
1.1 产品定位
“知礼”是一款基于用户画像、场景感知与情感计算的礼品推荐AI助手，帮助用户在送礼场景中快速做出得体、不出错的决策。核心解决“选礼纠结”“不了解对方喜好”“预算尴尬”“不懂场合分寸”等社交痛点。

1.2 目标用户
18-35岁年轻人，需要为伴侣、家人、朋友、同事、长辈等挑选礼物。

对社交礼仪敏感，希望规避风险、表达心意。

习惯使用微信生态，追求轻量、快捷的决策工具。

1.3 核心价值
懂礼：内置场合礼仪知识库，避免社交踩雷。

懂你：基于收礼人画像（关系、年龄、兴趣）个性化推荐。

有心：提供送礼记录、提醒、贺卡生成等情感陪伴功能。

1.4 产品形态
微信小程序 + 公众号服务号。小程序为核心工具，公众号负责内容触达、用户留存与提醒服务。

2. 用户场景与核心任务
场景	用户目标	任务流程
生日/纪念日送礼	快速找到合适且不踩雷的礼物	创建收礼人画像 → 选择场合/预算 → 浏览推荐 → 收藏/购买
节日送礼（母亲节等）	提前准备，避免遗忘	订阅节日提醒 → 接收公众号/模板消息 → 打开预填场景的小程序
道歉/感谢	表达诚意，但不确定分寸	选择“道歉”场合 → AI推荐温情/实用且不越界的礼物，附带贺卡建议
日常惊喜	无明确预算，想表达心意	使用“无理由”场合，推荐温暖或搞怪型礼品
团体送礼（后续）	公司团建、多人凑份子	进入“团体模式”，按预算+风格批量推荐
3. 功能需求
3.1 用户端（小程序）
F1 收礼人画像创建/编辑

必填：关系（伴侣/家人/朋友/同事/长辈/老师/客户/其他）、年龄段（5档）。

选填：兴趣圈层（10个，最多选3个，可跳过）、禁忌/偏好（如气味敏感、宗教禁忌）。

支持多个画像，默认当前展示一个，可快速切换。

F2 商品推荐流（首页）

双列瀑布流，展示商品图片、标题、价格、简短推荐理由。

顶部横向筛选条：场合（生日/纪念日/节日/感谢/道歉/无理由）、预算（<100/100-300/300-500/500-1000/1000+）、风格（实用/仪式/搞怪/温情）。

支持下拉刷新、上拉加载更多，筛选条件变更实时刷新（防抖500ms）。

无结果时显示空状态插画，并提供“清空筛选”按钮。

F3 礼物详情页

商品图片轮播（≥3张）。

展示价格、推荐理由卡片（2-3条，基于匹配因子动态生成）。

折叠规格参数，横向类似推荐。

底部操作栏：收藏❤️ + “去购买”按钮（跳转第三方电商联盟webview或小程序）。

F4 收藏与送礼记录

收藏：双列列表，左滑删除，支持详情跳转。

送礼记录：用户购买后确认“已购买”的记录（手动或电商回调），可删除。避免重复送礼。

F5 订阅提醒

用户可订阅节日（母亲节、情人节等）或自定义生日（关联收礼人）。

调用微信订阅消息API，在提醒日前3天、1天推送模板消息。

支持查看已订阅列表、取消订阅。

F6 个人中心

微信授权登录（头像、昵称）。

入口：我的收藏、送礼记录、订阅提醒、收礼人管理、设置（清空缓存、隐私协议等）。

游客模式：支持未登录使用，但收藏仅本地保存。

3.2 公众号服务号
菜单设计

开始选礼 → 跳小程序首页（路径带from=menu_start）

我的 → 跳小程序“我的”页面

灵感库 → H5文章聚合页（节日专题、热门榜单、真人故事）

自动回复

关注自动回复：文本 + 小程序卡片“开始选礼”

关键词回复：“送礼”“男友”“母亲节”等推送图文+对应场景的小程序卡片

模板消息

用于节日/生日提醒（需用户订阅）

内容示例：“母亲节还有7天，提前挑礼物更从容 [点击选礼]”

内容推送

每月4次，每篇文章内置3-5个小程序卡片（预填场景/预算）

3.3 后台管理系统
B1 数据看板

核心指标：DAU、推荐请求量、CTR、CVR、收藏率、订阅率。

漏斗分析：启动→完成推荐→详情→去购买→确认购买。

按收礼人关系、场合、价格带下钻。

B2 商品标签管理

商品池（MVP手动200个，后续自动化抓取）。

打标：基础标签（性别、年龄段、价格带）、兴趣圈层、情境标签（场合、风格）、禁忌标签。

支持批量导入/导出Excel。

B3 推荐配置

动态调整兴趣/情境/基础权重（初始0.5/0.3/0.2）。

探索率设置（如10%随机推荐）。

兜底策略：结果不足时填充热门商品。

B4 用户反馈处理

查看用户“不喜欢”反馈（商品ID、反馈类型、关联画像）。

支持人工标记处理，并可全局屏蔽某些商品。

B5 AB实验管理

创建实验组（如调整权重或新召回策略）。

查看实验组与对照组的CTR/CVR显著性对比。

4. 数据设计
4.1 用户画像标签体系
维度	取值	说明
关系	伴侣/家人/朋友/同事/长辈/老师/客户/其他	必填
年龄段	<18,18-25,26-35,36-45,46+	必填
兴趣圈层	科技数码、文艺生活、户外运动、美妆护肤、居家宅人、时尚穿搭、美食美酒、健康养生、职场办公、母婴亲子	最多3个，可跳过
禁忌	气味敏感、宗教禁忌等	可选
4.2 商品标签体系
每个商品打三类标签：

基础标签：适用性别、年龄段、价格带

兴趣标签：所属圈层（可多个）

情境标签：适用场合、风格（实用/仪式/搞怪/温情）、关键词

4.3 推荐匹配得分公式
text
总得分 = 兴趣匹配分×0.5 + 情境匹配分×0.3 + 基础匹配分×0.2
子项	计算规则
兴趣匹配分	重合圈层数：3个→10分，2个→7分，1个→4分，0个→0分；未选圈层→5分
情境匹配分	场合匹配（完全匹配6分/通用4分/不匹配0分）+ 风格匹配（完全4分/部分2分/冲突0分）
基础匹配分	性别匹配（匹配4分/通用3分/不匹配0分）+ 年龄匹配（完全3分/相邻1分/不匹配0分）+ 预算匹配（预算内3分/超≤20%得1分/超>20%0分）
4.4 推荐理由生成规则（可解释性）
根据得分最高的1-2个因子，从模板库生成文案：

因子	图标	文案模板
兴趣匹配	🎯	“因为ta喜欢{圈层名称}，{商品一句话卖点}。”
场合匹配	🎂	“{场合}选它，{场合关键词}。”
风格匹配	❤️	“{风格关键词}之选，表达心意恰到好处。”
预算匹配	💰	“刚好在{预算区间}内，性价比不错。”
禁忌规避	🚫	“特意避开{禁忌项}，这份礼物更安心。”
4.5 情感与礼仪知识（后续深度）
构建场合-关系-规则知识图谱（如道歉场合下，禁止搞怪风格，提升温情权重）。

引入送礼动机分类：表达爱意、维护关系、互惠回报、补偿心理、展示身份。

通过轻量问卷收集用户情绪（焦虑度、期待值），调整推荐策略（高焦虑用户优先推荐高成功率经典款）。

7. 电商CPS与归因
7.1 接入平台
京东联盟（优先）：小程序跳转体验好，自营品质保障，佣金5%-30%。

淘宝联盟（补充）：商品库最全，需处理跳转损耗，佣金5%-30%。

多多进宝（可选）：低价小礼品，佣金1%-50%。

7.2 归因技术要点
为每个用户生成带推广位PID的专属转链URL。

使用relationId（京东）或rid（淘宝）参数实现渠道归因。

在小程序端通过webview打开H5推广页，保留追踪参数。

后台调用联盟订单API核对tk_status（13为已结算）。

7.3 收入预估（MVP）
假设平均佣金率10%，客单价150元，单次佣金15元。

日活2000，购买转化率5%，日单量100，日佣金1500元，月收入约4.5万元。

扣除平台服务费10%及运营成本，月净利约2-3万元。

8. 运营与商业化
8.1 商业模式（MVP阶段）
CPS佣金：主要收入来源。

后续可选：会员订阅（无限收藏、人工顾问）、品牌直接合作（一件代发）、定制礼品服务。

8.2 增长策略
内部种子：亲友使用，朋友圈推广。

内容引流：小红书笔记“送礼AI测评”、公众号节日攻略。

裂变：送礼海报、邀请有礼（v1.2）。

线下：与礼品店合作放置小程序码（远期）。

8.3 节日运营日历
提前3个月规划：情人节、母亲节、父亲节、七夕、圣诞、春节。每个节点提前2周启动公众号推文+小程序专属卡片。

9. 风险与合规
9.1 合规要求
隐私政策：明确告知收集信息仅为关系、年龄段等非识别信息，不存储真实姓名/电话。

订阅消息：每次授权需明确用途，不可滥用（不能发送营销内容）。

电商跳转：需提示“即将离开小程序，第三方平台负责交易”。

9.2 风险预案
推荐不准：后台提供“人工精选榜单”兜底。

转化率低：与品牌方合作提升佣金，或尝试自建商城（长期）。

小程序被拒：预备好电商平台资质、隐私审核材料。
